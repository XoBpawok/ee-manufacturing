# Дизайн: EVE Echoes Manufacturing Calculator

Дата: 2026-06-16

## Мета

Веб-сторінка для розрахунку необхідних ресурсів, їх вартості та часу побудови предметів у EVE Echoes.
Перший цільовий предмет — корабель **Naglfar**. Сторінка універсальна: будь-який craftable-предмет обирається зі списку.

## Вимоги користувача

- Бачити, що саме потрібно для побудови предмета (дерево матеріалів).
- Ціна за одиницю матеріалу та сума за матеріал.
- Підсумки по категоріях товарів.
- Вартість (`manufacture_cost`) та час (`manufacture_time`) створення кожного елемента.
- Набір індустрі-скілів, потрібних для крафту, з можливістю змінювати рівні й бачити вплив на кількість та вартість.
- Універсальний вибір предмета зі списку.

## Стек

Vite + React + TypeScript + Ant Design. Жива загрузка з API echoes.mobi, кеш у `localStorage` з TTL (24 год) + кнопка «оновити».

## Архітектура

```
src/
  api/        — фетчери ендпоінтів + TS-типи DTO; нормалізація (string→number)
  domain/     — чисті функції: побудова дерева крафту, рушій вартості, рушій скілів, агрегації
  components/ — ItemSelector, CraftTree, SkillsPanel, SummaryPanel
  store/      — стан застосунку (вибраний предмет, build/buy мапа, рівні скілів, ручні ціни)
  App.tsx
```

Бізнес-логіка (`domain/`) — чисті функції з юніт-тестами, не залежать від React/UI.

### Шар даних

При відкритті завантажуються 4 ресурси (`Accept: application/json`, браузерний User-Agent):
- `/api/items` — предмети (селектор, назви, іконки)
- `/api/v2/item_blueprints` — рецепти (≈1528, одним запитом)
- `/api/v2/item_prices` — `estimated_price` (6151 предмет)
- `/api/v2/industry_skills` — `efficiency`, `time` по рівнях 1-5

Будуються мапи: `itemById`, `blueprintByItemId`, `priceByItemId`, `skillByName`.
Кеш у `localStorage` з TTL; кнопка примусового оновлення.

### Селектор предмета

Пошуковий dropdown (Ant `Select` з `showSearch`) з усіх предметів, що мають блюпрінт (`blueprintByItemId`).
Дефолт — Naglfar (`item_id` 10701000201).

### Рекурсивне дерево build-vs-buy

Корінь = обраний предмет (режим build). Кожен матеріал має перемикач **купити / крафтити**:
- *купити* → `ціна × кількість` (`estimated_price` або ручна ціна)
- *крафтити* → розкриває під-матеріали блюпрінта + додає `manufacture_cost` (вартість job) та `manufacture_time`
- Матеріал без блюпрінта (лист, напр. мінерал) — завжди «купити».
- Дефолт: прямі компоненти кореня — у режимі «купити»; користувач сам розкриває глибше.
- Захист від циклів у рекурсії (visited-set за `item_id`).

UI — Ant `Table` з розгортанням рядків (`expandable`) або деревоподібна таблиця: назва, тип/категорія, кількість, ціна за одиницю (редагована), сума, кнопка перемикача build/buy.

### Рушій скілів та вартості

`efficiency`-скіли знижують **кількість** матеріалів, `time`-скіли знижують **час** job.
Кількості в блюпрінті задані для **максимальних** скілів (рівень 5).

Для блюпрінта з релевантними скілами (`blueprint.skills`):
```
qty(levels) = ceil( qty_bp × (1 − Σeff(levels)/100) / (1 − Σeff_max/100) )
```
- На рівні 5 усіх скілів `qty(levels) == qty_bp` (дефолт точно збігається з блюпрінтом).
- Зниження рівня збільшує кількість.
- Час job: `time(levels) = manufacture_time × Π(1 + time_factor(level_i))` (множники з поля `time`).

**Припущення:** комбінування скілів **адитивне** (сума %). Виноситься в окрему чисту функцію `combineEfficiency()` з тестами — легко замінити на мультиплікативну модель. Оскільки дефолт = макс рівні, на стартовому вигляді припущення не впливає.

### Панель скілів

Усі індустрі-скіли, задіяні в поточному дереві (обʼєднання `blueprint.skills` усіх блюпрінтів у дереві), кожен зі слайдером рівня 0-5 (Ant `Slider`), дефолт 5.
Зміна слайдера миттєво перераховує кількості, вартість і час (реактивний стан).

### Підсумки (SummaryPanel)

- Таблиця матеріалів: назва, тип/категорія, кількість (з урахуванням скілів), ціна за одиницю (редагована), сума.
- Підсумки по категоріях товарів (групування за `type`/`category_name`).
- Загальна вартість матеріалів + сумарна вартість job (Σ `manufacture_cost` усіх built-вузлів) + загальний час.
- Порівняння **зкрафтити vs купити готовий** (`estimated_price` кінцевого предмета).
- Форматування чисел як ISK.

## Обробка помилок

- Збій фетчу → повідомлення + кнопка повтору; за наявності — показ кешованих даних.
- Предмет без блюпрінта → лист дерева, режим тільки «купити».
- Відсутня ціна матеріалу → 0 + візуальна позначка «ціна невідома», поле редаговане.

## Тестування

Юніт-тести (Vitest) на `domain/`:
- `combineEfficiency` (адитивна модель, межі 0 і 5)
- `effectiveQuantity` (рівень 5 == блюпрінт; нижчі рівні збільшують)
- побудова дерева (рекурсія, захист від циклів, build vs buy)
- агрегації (по категоріях, сумарна вартість, час)
