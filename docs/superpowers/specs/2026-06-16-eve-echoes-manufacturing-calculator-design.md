# Дизайн: EVE Echoes Manufacturing Calculator

Дата: 2026-06-16 (оновлено: додано авто-оптимізацію, реверс-інжиніринг, іконки)

## Що це й навіщо

Веб-сторінка, яка показує, **що, скільки й за яку ціну** потрібно, щоб побудувати предмет у EVE Echoes, і **скільки це коштує та триває**. Перший цільовий предмет — корабель **Naglfar**, але сторінка універсальна: будь-який craftable-предмет обирається зі списку.

Головна цінність: предмет будується **рекурсивно** (корабель → компоненти → мінерали), і для кожного компонента є вибір — **купити** його на ринку чи **зкрафтити** самому. Інструмент рахує сумарну вартість для будь-якої комбінації цих рішень і може **автоматично знайти найдешевшу**.

## Джерело даних

Жива загрузка з [echoes.mobi API](https://echoes.mobi/api) (деталі — у `CLAUDE.md`). Завантажуються 4 ендпоінти, кешуються в `localStorage` на 24 год:

1. `item_blueprints` — рецепти виробництва
2. `item_reverse_engineering` — рецепти реверс-інжинірингу (з `pass_rate`)
3. `item_prices` — `estimated_price` + `icon_id`
4. `industry_skills` — `efficiency` / `time` по рівнях 1-5

## Ключові поняття

### Рецепт (Recipe)
Виробництво й реверс мають **однакову структуру** (матеріали, вартість job, час, скіли) і **не перетинаються** (предмет має максимум один рецепт). Тому вони злиті в одну мапу `recipeByItemId` з полем `kind`. Реверс додатково має `passRate < 1`.

### Дерево крафту
Корінь — обраний предмет. Кожен матеріал — це вузол, який можна **купити** (лист) або **крафтити** (розкривається у власні матеріали). Лист без рецепта (мінерал) — завжди купівля. Рекурсія захищена від циклів.

### Режими вузла
- **buy** — вартість = `кількість × ціна` (estimated_price або введена вручну)
- **build (manufacture)** — `runs = ceil(потрібно / output_number)`; матеріали зменшені скілами efficiency; вартість job = `manufacture_cost × runs`
- **build (reverse)** — те саме, але `attempts = runs / passRate` (очікувані спроби); матеріали, вартість і час множаться на `attempts`; скіли efficiency **не** діють на реверс

## Як працюють скіли (efficiency)

Кількості в рецепті задані для **максимальних** скілів (рівень 5). Зворотна формула масштабує їх для обраних рівнів:

```
кількість = ceil( кількість_рецепта × (1 − effCur/100) / (1 − effMax/100) )
```

- `effMax` — сума efficiency% релевантних скілів на рівні 5
- `effCur` — сума на поточних рівнях (слайдери)
- на рівні 5 → точно як у рецепті; нижче → більше матеріалів
- комбінування **адитивне** (припущення; винесено в `combineEfficiency`, легко змінити)
- `time`-скіли масштабують час job так само; `manufacture_cost` скілами не змінюється

Панель скілів показує лише ті скіли, що задіяні у вузлах режиму **build** (з'являються в міру розкриття дерева).

## Авто-оптимізація

Перемикач «Авто-оптимізація». Коли увімкнено, для **кожного** предмета рекурсивно обчислюється найдешевше джерело (`computeOptimalBuildSet`):

```
unitCost(item) = min(
  ціна купівлі,
  (manufacture_cost + Σ unitCost(дочірній) × кількість) / (output_number × passRate)
)
```

З мемоізацією за `itemId` і захистом від циклів. Для рішення використовуються неперервні (без округлення) вартості за одиницю; efficiency-скіли та passRate враховані. Результат — набір `itemId`, які вигідніше крафтити; ним наповнюється `buildSet`, і дерево перебудовується. Ручні перемикачі при цьому вимкнені.

> Перевірено на Naglfar: авто-режим обрав крафт 10 з 13 компонентів, знизивши вартість з 33.6 млрд до 25.9 млрд ISK (економія ~11.9 млрд проти купівлі готового).

## Архітектура коду

```
src/
  api/        client.ts (фетч+нормалізація+кеш), types.ts (доменні типи, iconUrl)
  domain/     skills.ts, tree.ts (дерево+агрегації), optimize.ts (авто-вибір), format.ts
              + юніт-тести (skills, tree, optimize)
  components/ ItemSelector, SkillsPanel, CraftTree, SummaryPanel, ItemIcon
  store/      useCalculator — стан і похідні (buildSet, дерево, підсумки)
  App.tsx
```

Уся бізнес-логіка (`domain/`) — чисті функції з юніт-тестами, незалежні від React.

## UI

- **Панель керування:** селектор предмета (пошук), кількість, перемикач авто-оптимізації, кнопка «Оновити дані».
- **Скіли:** слайдери рівнів 0-5, кнопка «усі на макс».
- **Підсумки:** картки (разом / матеріали / job / час / купити готовий / економія), список покупок (агреговано, з іконками й редагованою ціною), підсумки по категоріях, перелік jobs (вартість, час, тег «реверс»).
- **Дерево крафту:** розгортувана таблиця з іконками, тегами режиму (купити / крафт ×N / реверс ×N), редагованою ціною та перемикачем режиму.

## Обробка помилок

Збій фетчу → повідомлення + повтор. Предмет без рецепта → лист (тільки купити). Невідома ціна → 0 + позначка «ціна невідома», поле редаговане.
